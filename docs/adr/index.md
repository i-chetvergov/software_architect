### ADR — Architecture Decision Records

В данном разделе фиксируются ключевые архитектурные решения по платформе унифицированных коммуникаций. ADR служат связующим звеном между архитектурными драйверами, требованиями, диаграммами и реализованными решениями.

#### Оглавление ADR

- **ADR-001 — Архитектурный стиль и границы сервисов (DDD/Bounded Contexts)**  
- **ADR-002 — Синхронные и асинхронные коммуникации (REST/gRPC + event bus)**  
- **ADR-003 — Выбор хранилищ (OLTP/NoSQL/Time-series) и распределение данных**  
- **ADR-004 — IAM/OIDC, сервисные токены и mTLS между сервисами**  
- **ADR-005 — Observability (логи/метрики/трейсы) и корреляция вызовов**  
- **ADR-006 — HA/Failover стратегии и SLO для UC-платформы**  
- **ADR-007 — API Gateway и стратегия версионирования API**  
- **ADR-008 — Управление конфигурацией и секретами**  
- **ADR-009 — Интеграция SIP/WebRTC и медиа-плоскости**  
- **ADR-010 — Стратегия данных CDR/Recording/Retention**

Каждый ADR размещён в отдельном файле `ADR-XXX.md` в данном каталоге и содержит разделы: **Статус**, **Контекст**, **Решение**, **Альтернативы**, **Последствия**, а также ссылки на соответствующие архитектурные драйверы, диаграммы (`docs/10_diagrams/*.puml`) и разделы архитектурного описания.
