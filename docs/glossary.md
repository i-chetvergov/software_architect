# Глоссарий терминов Unified Communications

---

| Термин | Сокращение | Описание |
|------|------------|----------|
| Unified Communications | UC | Класс корпоративных информационных платформ, объединяющих сервисы голосовой связи, видеосвязи, обмена сообщениями, конференций, присутствия и интеграций в единой программной среде и интегрируемых с бизнес-процессами организации. |
| UC Platform | — | Корпоративная программная платформа, реализующая концепцию Unified Communications и предоставляющая централизованные сервисы коммуникации, управления и интеграции. |
| UC Client | — | Клиентское приложение пользователя (Web, Desktop, Mobile), обеспечивающее доступ к сервисам UC-платформы. |
| Session Initiation Protocol | SIP | Протокол сигнализации прикладного уровня, используемый для установления, модификации и завершения мультимедийных сеансов связи (голос, видео, сообщения) в IP-сетях. |
| Web Real-Time Communication | WebRTC | Набор стандартов и технологий для защищённой передачи аудио-, видео- и данных в реальном времени между браузерами и приложениями с использованием DTLS, SRTP и ICE. |
| Real-time Transport Protocol | RTP | Протокол передачи медиапотоков в реальном времени, используемый для доставки аудио и видео данных. |
| Secure RTP | SRTP | Расширение RTP, обеспечивающее шифрование, аутентификацию и целостность медиапотоков. |
| Session Border Controller | SBC | Пограничный сетевой элемент, обеспечивающий защиту, контроль и маршрутизацию SIP- и медиатрафика на границе корпоративной сети (NAT traversal, topology hiding, security). |
| Softswitch | — | Программный коммутатор, выполняющий функции сигнализации и управления соединениями между абонентами UC-платформы, ВКС-серверами и телефонной сетью общего пользования. |
| Call Control | — | Логический слой UC-платформы, отвечающий за обработку вызовов: маршрутизацию, правила набора, очереди, переводы, удержание, парковку и управление конференциями. |
| Video Conferencing Server | VCS / ВКСС | Сервер видеоконференцсвязи, обеспечивающий проведение индивидуальных и групповых конференций, управление ролями участников, запись и подключение внешних абонентов. |
| Presence | — | Подсистема UC-платформы, собирающая и распространяющая информацию о статусе доступности пользователей (доступен, занят, отсутствует и т.д.). |
| Messaging / Chat | — | Сервисы обмена текстовыми и мультимедийными сообщениями, включая личные и групповые чаты, каналы, треды и интеграции с внешними системами. |
| Group Chat | — | Механизм обмена сообщениями между несколькими пользователями в рамках одного логического пространства общения. |
| Media Services | — | Компоненты платформы, обрабатывающие медиапотоки (RTP/SRTP), включая микширование, транскодирование, запись и воспроизведение. |
| Call Recording | — | Функциональность записи голосовых и видеосеансов с возможностью последующего хранения, поиска и воспроизведения. |
| Provisioning | — | Совокупность процессов и сервисов, обеспечивающих жизненный цикл пользователей, устройств, номеров и конфигураций UC-платформы. |
| Call Detail Records | CDR | Подсистема сбора и хранения детализированной информации о сеансах связи (время, длительность, участники, ресурсы) для отчётности и аналитики. |
| Notification Service | — | Сервис доставки уведомлений о событиях платформы через различные каналы (email, SMS, push-уведомления, мессенджеры). |
| API Gateway | — | Компонент, предоставляющий единую точку входа для внешних клиентов и интеграций, выполняющий аутентификацию, авторизацию, маршрутизацию и лимитирование API-запросов. |
| Identity and Access Management | IAM | Подсистема управления учётными записями, аутентификацией и авторизацией пользователей и сервисов, включая реализацию RBAC/ABAC и интеграцию с корпоративными каталогами. |
| Role-Based Access Control | RBAC | Модель управления доступом, основанная на ролях пользователей. |
| Attribute-Based Access Control | ABAC | Модель управления доступом, основанная на атрибутах субъектов, ресурсов и контекста. |
| Multi-Tenancy | — | Архитектурный принцип, позволяющий обслуживать несколько организаций (тенантов) в рамках одной UC-платформы с логической изоляцией данных. |
| Integration Plugin | — | Модуль интеграции UC-платформы с внешними системами (CRM, Service Desk, Issue Tracker, сторонние мессенджеры). |
| Bot / Automation | — | Программный агент, взаимодействующий с пользователями UC-платформы через сообщения и команды для автоматизации бизнес-процессов. |
| Message Broker | — | Инфраструктурный компонент асинхронного обмена сообщениями между микросервисами. |
| Observability | — | Способность системы предоставлять метрики, логи и трассировки для мониторинга, диагностики и анализа работы системы. |
| Service Level Agreement | SLA | Формализованное соглашение об уровне предоставляемых сервисов, включая показатели доступности и производительности. |
| High Availability | HA | Архитектурный подход, обеспечивающий непрерывность работы системы при отказе отдельных компонентов. |
| Disaster Recovery | DR | Набор мер и архитектурных решений для восстановления работоспособности системы после катастрофических сбоев. |
| Representational State Transfer | REST | Архитектурный стиль взаимодействия компонентов поверх HTTP, применяемый для построения внешних и публичных API. |
| gRPC | gRPC | Фреймворк для удалённых вызовов процедур, использующий Protocol Buffers и HTTP/2, применяемый для высокопроизводительного взаимодействия внутренних сервисов. |
| Apache Kafka | Kafka | Распределённая платформа потоковой передачи и хранения событий, используемая для построения event-driven и streaming архитектур. |
| RabbitMQ | — | Брокер сообщений, реализующий протокол AMQP, используемый для асинхронного обмена сообщениями и событий между микросервисами. |
| Advanced Message Queuing Protocol | AMQP | Открытый стандартный протокол обмена сообщениями для асинхронной коммуникации между сервисами. |
| Event Bus / Event Stream | — | Инфраструктурный компонент для публикации и подписки на доменные события в event-driven архитектуре. |
| Schema Registry | — | Хранилище схем сообщений (Avro, JSON Schema, Protobuf), обеспечивающее совместимость версий при асинхронной коммуникации. |
| PostgreSQL | PostgreSQL | Реляционная система управления базами данных (OLTP), используемая для хранения транзакционных данных платформы. |
| Redis | — | In-memory хранилище структурированных данных, используемое для кэширования и управления сессиями. |
| Object Storage | — | Хранилище неструктурированных данных (S3-совместимое), используемое для медиафайлов и записей разговоров. |
| Time-Series Database | TSDB | Специализированная база данных, оптимизированная для хранения и запросов временных рядов (метрики, логи, трейсы). |
| Columnar Database | — | База данных с колоночным хранением, оптимизированная для аналитических запросов по большим объёмам данных (CDR, отчётность). |
| Online Transaction Processing | OLTP | Тип обработки данных, ориентированный на быстрые транзакционные операции с высокой частотой чтения/записи. |
| Online Analytical Processing | OLAP | Тип обработки данных, ориентированный на аналитические запросы по большим объёмам исторических данных. |
| NoSQL | — | Класс систем управления базами данных, не использующих реляционную модель данных. |
| WebSocket | — | Протокол двусторонней связи поверх TCP, используемый для real-time коммуникации между клиентами и серверами. |
| OAuth 2.1 | OAuth 2.1 | Протокол авторизации для предоставления третьим сторонам ограниченного доступа к защищённым ресурсам. |
| OpenID Connect | OIDC | Протокол аутентификации на основе OAuth 2.0, используемый для идентификации пользователей через ID-токены. |
| Mutual Transport Layer Security | mTLS | Расширение TLS, обеспечивающее взаимную аутентификацию клиента и сервера, используемое для межсервисной коммуникации. |
| Transport Layer Security | TLS | Криптографический протокол, обеспечивающий защищённую передачу данных между клиентами и серверами. |
| Domain-Driven Design | DDD | Методология проектирования программных систем, основанная на моделировании предметной области и выделении bounded contexts. |
| Bounded Context | — | Явная граница в предметной области, внутри которой действует единая модель данных и терминология. |
| Event-Driven Architecture | EDA | Архитектурный паттерн, основанный на асинхронном обмене событиями между слабосвязанными компонентами системы. |
| Event Sourcing | — | Архитектурный паттерн, предполагающий хранение всех изменений состояния системы как последовательности событий. |
| Command Query Responsibility Segregation | CQRS | Архитектурный паттерн, разделяющий операции записи и чтения на отдельные модели данных. |
| Microservices Architecture | — | Архитектурный стиль, при котором система разбита на независимые сервисы с чёткими границами ответственности. |
| Database per Service | — | Принцип микросервисной архитектуры, при котором каждый сервис имеет собственное хранилище данных. |
| Kubernetes | K8s | Система оркестрации контейнеров для автоматизации развёртывания, масштабирования и управления контейнеризованными приложениями. |
| Docker | — | Платформа для разработки, доставки и запуска приложений в изолированных контейнерах. |
| Service Mesh | — | Инфраструктурный слой для управления межсервисной коммуникацией (маршрутизация, безопасность, наблюдаемость). |
| Container Orchestration | — | Автоматизация управления жизненным циклом контейнеров: развёртывание, масштабирование, балансировка нагрузки и отказоустойчивость. |
