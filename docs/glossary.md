# Глоссарий терминов Unified Communications

---

| Термин | Сокращение | Описание |
|------|------------|----------|
| Unified Communications | UC | Класс корпоративных информационных платформ, объединяющих сервисы голосовой связи, видеосвязи, обмена сообщениями, конференций и присутствия в единой программной среде и интегрируемых с бизнес‑процессами организации. |
| Session Initiation Protocol | SIP | Протокол сигнализации прикладного уровня, используемый для установления, модификации и завершения мультимедийных сеансов связи (голос, видео, сообщения) в IP‑сетях. |
| Web Real-Time Communication | WebRTC | Набор стандартов и технологий для организации защищённой передачи аудио‑, видео‑ и данных в реальном времени непосредственно между браузерами и приложениями без установки дополнительного ПО. |
| Session Border Controller | SBC | Сетевой элемент, обеспечивающий контроль, защиту и маршрутизацию SIP‑ и медиатрафика на границе корпоративной или операторской сети (NAT traversal, topology hiding, security). |
| Call Control | — | Логический слой UC‑платформы, отвечающий за обработку вызовов: маршрутизацию, правила набора, очереди, переводы, удержание, парковку и управление конференциями. |
| Presence | — | Подсистема UC‑платформы, собирающая и распространяющая информацию о статусе доступности пользователей (доступен, занят, отсутствует и т.д.) между клиентами и интеграциями. |
| Messaging / Chat | — | Сервисы обмена текстовыми и мультимедийными сообщениями, включая личные и групповые чаты, чаты конференций и интеграции с внешними каналами связи. |
| Media / Recording | — | Компоненты платформы, обрабатывающие медиапотоки RTP/SRTP, включая микширование, транскодирование, запись, хранение и предоставление записей разговоров и конференций. |
| Provisioning | — | Совокупность процессов и сервисов, обеспечивающих жизненный цикл пользователей, устройств, номеров и конфигураций UC‑платформы. |
| Call Detail Records | CDR | Подсистема сбора и хранения детализированной информации о сеансах связи (время, длительность, участники, ресурсы), используемой для биллинга, отчётности и аналитики. |
| Notification Service | — | Сервис доставки уведомлений о событиях платформы и бизнес‑процессах через различные каналы (email, SMS, push‑уведомления, мессенджеры). |
| API Gateway | — | Компонент, предоставляющий единую точку входа для внешних клиентов и интеграций, выполняющий маршрутизацию запросов, аутентификацию, авторизацию, лимитирование и версионирование API. |
| Identity and Access Management | IAM | Подсистема управления учётными записями, аутентификацией и авторизацией пользователей и сервисов, включая интеграцию с корпоративными каталогами и реализацию OAuth 2.1 / OpenID Connect. |
| Quality of Service | QoS | Совокупность механизмов и параметров, обеспечивающих гарантированные характеристики качества передачи медиатрафика (задержка, потери, джиттер). |
| Observability | — | Способность системы предоставлять метрики, логи и трассировки, необходимые для мониторинга, диагностики инцидентов и оценки соблюдения SLO/SLA. |
