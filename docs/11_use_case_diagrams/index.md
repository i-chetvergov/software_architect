# Use Case диаграммы

**Проект:** Корпоративная платформа унифицированных коммуникаций  
(**Unified Communications, UC**)

---

## Введение

Настоящий раздел содержит Use Case диаграммы (UML Use Case), описывающие функциональные возможности UC-платформы с точки зрения различных акторов системы.

Use Case диаграммы показывают:
- **Акторов** — роли пользователей и внешних систем, взаимодействующих с платформой;
- **Use Cases** — функциональные возможности, которые система предоставляет акторам;
- **Связи** между акторами и Use Cases (ассоциации, зависимости, включения, расширения).

Диаграммы организованы по акторам для лучшей читаемости и структурирования функциональных требований.

---

## Акторы системы

В UC-платформе выделены следующие основные акторы:

1. **Корпоративный пользователь** — сотрудник, агент, использующий UC-платформу для ежедневной работы
2. **Внешний пользователь** — любой человек, подключающийся по ссылке через приложение или браузер
3. **Администратор UC** — IT-администратор, управляющий системой, пользователями и конфигурацией
4. **Офицер информационной безопасности** — специалист по безопасности, контролирующий доступ и политики ИБ

## Связь с другими разделами

Use Case диаграммы связаны с:

- **Функциональными требованиями (FR)** — каждый Use Case покрывает одно или несколько требований из раздела «Требования»;
- **Диаграммами последовательности** — детальные sequence диаграммы в разделе «Диаграммы последовательности сценариев» показывают техническую реализацию Use Cases;
- **Архитектурными решениями** — Use Cases определяют границы ответственности компонентов системы;
- **Сценариями атрибутов качества (QAS)** — Use Cases проверяются через QAS для валидации NFR.

## Трассируемость

Трассируемость Use Cases обеспечивается через:

- Use Case → Функциональные требования (FR) → Архитектурные драйверы (AD) → ADR
- Use Case → Sequence диаграммы → Компоненты C4 Containers/Deployment
- Use Case → QAS → NFR → SLO/SLA
