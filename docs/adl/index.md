# ADL — Architecture Decision Log

В данном разделе фиксируются ключевые архитектурные решения по платформе унифицированных коммуникаций. ADR служат связующим звеном между архитектурными драйверами, требованиями, диаграммами и реализованными решениями.

## Оглавление ADR

- [**ADR-001 — Архитектурный стиль и границы сервисов (DDD/Bounded Contexts)**](../adr/ADR-001.md)
- [**ADR-002 — Синхронные и асинхронные коммуникации (REST/gRPC + event bus)**](../adr/ADR-002.md)
- [**ADR-003 — Выбор хранилищ (OLTP/NoSQL/Time-series) и распределение данных**](../adr/ADR-003.md)
- [**ADR-004 — IAM/OIDC, сервисные токены и mTLS между сервисами**](../adr/ADR-004.md)
- [**ADR-005 — Observability (логи/метрики/трейсы) и корреляция вызовов**](../adr/ADR-005.md)
- [**ADR-006 — HA/Failover стратегии и SLO для UC-платформы**](../adr/ADR-006.md)
- [**ADR-007 — API Gateway и стратегия версионирования API**](../adr/ADR-007.md)
- [**ADR-008 — Управление конфигурацией и секретами**](../adr/ADR-008.md)
- [**ADR-009 — Интеграция SIP/WebRTC и медиа-плоскости**](../adr/ADR-009.md)
- [**ADR-010 — Стратегия данных CDR/Recording/Retention**](../adr/ADR-010.md)

Каждый ADR размещён в отдельном файле `ADR-XXX.md` в каталоге ADR и содержит разделы: **Статус**, **Контекст**, **Решение**, **Альтернативы**, **Последствия**, а также ссылки на соответствующие архитектурные драйверы, диаграммы (`docs/10_diagrams/*.puml`) и разделы архитектурного описания.
